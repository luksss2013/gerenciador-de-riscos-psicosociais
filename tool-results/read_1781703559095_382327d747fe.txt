     5‚Üí## Project Context
     6‚Üí
     7‚ÜíImplementing a multi-tenant SaaS for psychosocial occupational risk management per Brazilian NR-1 regulation, using the COPSOQ II-BR instrument (40 items, 11 dimensions, Likert 5-point scale).
     8‚Üí
     9‚Üí### Adaptations to sandbox constraints
    10‚Üí
    11‚Üí- Single Next.js 16 app (no monorepo / apps+packages split).
    12‚Üí- SQLite via Prisma (no Neon/PostgreSQL RLS ‚Äî tenant isolation enforced at app layer via `professionalId`).
    13‚Üí- All views mounted under `/` (single route). Internal navigation via Zustand store view-state.
    14‚Üí- Worker portal simulated in-app (demo mode) ‚Äî same `/` route, view-state driven ‚Äî preserves the privacy contract conceptually (no PII collected from worker).
    15‚Üí- Reports: in-app HTML preview + metadata persistence (no R2/PDF/DOCX binary generation in sandbox).
    16‚Üí- Auth: lightweight session cookie via httpOnly cookie + bcrypt-style hash (no Better Auth dependency).
