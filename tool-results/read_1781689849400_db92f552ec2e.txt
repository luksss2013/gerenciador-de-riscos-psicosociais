     5→## Project Context
     6→
     7→Implementing a multi-tenant SaaS for psychosocial occupational risk management per Brazilian NR-1 regulation, using the COPSOQ II-BR instrument (40 items, 11 dimensions, Likert 5-point scale).
     8→
     9→### Adaptations to sandbox constraints
    10→
    11→- Single Next.js 16 app (no monorepo / apps+packages split).
    12→- SQLite via Prisma (no Neon/PostgreSQL RLS — tenant isolation enforced at app layer via `professionalId`).
    13→- All views mounted under `/` (single route). Internal navigation via Zustand store view-state.
    14→- Worker portal simulated in-app (demo mode) — same `/` route, view-state driven — preserves the privacy contract conceptually (no PII collected from worker).
    15→- Reports: in-app HTML preview + metadata persistence (no R2/PDF/DOCX binary generation in sandbox).
    16→- Auth: lightweight session cookie via httpOnly cookie + bcrypt-style hash (no Better Auth dependency).

    73→Key decisions:
    74→- Single-route constraint: worker portal returns `{token, redirectUrl: '/?worker='+token}` (query-string routing) instead of HTTP 302 redirect to `/q/:token`. Frontend will read `?worker=` query and render worker UI.
    75→- Trend route param renamed from `:companyId` (spec contract) to `[id]` (Next.js requires slug-name uniformity across sibling dynamic segments under `/companies/[id]/*`). API path/contract semantics preserved.
    76→- All `Assessment` tenant checks go via `requireTenantOwnership(assessment.professionalId, current.id)`; company checks via `company.professionalId`.
    77→- Cookie format: `nr1_session=<token>; HttpOnly; Path=/; Max-Age=604800; SameSite=Strict`. Logout clears via Max-Age=0.
    78→- All worker portal routes use `void token` or `void params` where appropriate to satisfy Next.js route signature; no session cookies, no Set-Cookie headers (RB-03 anti-fingerprinting).
    79→- All route handlers wrapped in try/catch; unhandled errors return errorJson(INTERNAL_ERROR, "Internal error"). Tenant/auth errors detected via `e.code` and re-thrown as proper errorJson.
